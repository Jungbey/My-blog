    使用Hexo搭建博客

    相关链接🔗： 安装Hexo、将 Hexo 部署到 GitHub Pages

    安装 Hexo

    这里只需要直接根据第一个链接提供的指令安装即可：

    将hexo添加到环境变量

    至此，hexo已经安装完成，可以在cmd中执行hexo的相关指令

    创建 Hexo 项目

    安装完成之后，执行以下指令，即可创建一个hexo项目

    配置 Hexo

    hexo项目的配置信息都在根目录中的 _config.yml 中，可以在此更改配置参数

    具体的配置参数可以在此查看：Hexo配置

    在此，我主要设置了一下网站的标题以、作者名、文章的 永久链接 格式及网站链接。

    书写文章

    执行以下命令来创建一篇新的文章

    • layout 为文章的布局，Hexo有三种默认布局： post、page和draft，创建文章时，会根据布局的不同而将文件放入不同的文件夹。layout 可以不填，默认为post，在创建时存入source/_post 目录。
    布局 路径
    post source/_posts
    page source/
    draft source/_drafts
    • title 即为文章的标题，为必须项

    创建文章后，会在相应的文件夹中新建对应的md文件，文件中会有默认的内容来保存文章的相关信息, 分别为 title、date、及tags.

    • title：即为文章的标题，默认为创建文章时所设定的标题
    • date：创建文章的日期
    • tags：文章的标签

    可以通过修改以上内容来修改文章的信息

    然后，就可以在下面编辑自己的文章内容进行创作啦 🎉

    一些Hexo的指令

    这里只列出一些我认为必须要知道的一些指令，更多指令可以在此查看： Hexo指令

    init

    $ hexo init [folder]

    新建一个Hexo项目，即一个博客网站，folder为文件夹的名称，如果没有设置，Hexo会默认在当前的文件夹建立项目。

    new

    $ hexo new [layout] </code></pre><p>新建一篇文章，如果文章的标题中有空格，则需要用引号括起来</p>

<pre><code>$ hexo g</code></pre><p>此指令可以生成静态文件，保存在<code>public/</code>目录中</p>

<p>参考链接：<a href="https://hexo.io/zh-cn/docs/github-pages" target="_blank" rel="noopener">将 Hexo 部署到 GitHub Pages</a></p>

<p>这里主要就是根据上面链接中的教程一步一步进行，为了更加方便，会使用到 <a href="https://travis-ci.com/" target="_blank" rel="noopener">Travis CI</a>，可以直接使用github账号注册</p>
<p><code>部署</code> 的大致步骤为：</p>
<ol>
<li>在 <a href="https://github.com/" target="_blank" rel="noopener">github</a> 中新建一个repository并将项目推送到该repository中；</li>
<li>在 <a href="https://github.com/marketplace/travis-ci" target="_blank" rel="noopener">这里 </a> 将 Travis CI 添加到 github账户中；</li>
<li>前往 github 中的 <a href="https://github.com/settings/installations" target="_blank" rel="noopener">Applications settings</a> 给予Travis CI 访问你的repository 的权限；</li>
<li>在 github 中  <a href="https://github.com/settings/tokens" target="_blank" rel="noopener">新建 Personal Access Token</a>， 只勾选 <code>repo</code> 权限并生成Token，生成后复制该Token；</li>
<li>在 <a href="https://travis-ci.com/" target="_blank" rel="noopener">这里</a> 找到并前往 相应的 repository 的 <code>设置页面</code>，在<strong>Environment Variables</strong>下新建一个环境变量，<strong>Name</strong>设置为<code>GH_TOKEN</code>, <strong>Value</strong>为之前复制的Token，然后点击<strong>Add</strong>保存；<blockquote>
<p><code>设置(settings)</code>在打开的repository管理页面右边的 More options 中, 点击即可进入<code>设置页面</code> </p>
</blockquote>
</li>
